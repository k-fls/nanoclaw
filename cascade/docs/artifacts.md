# Persistent surface

Everything not listed here is derived from git at runtime.

## Committed files

| Path | Role | Who writes | Who reads |
|---|---|---|---|
| `.cascade/branch-classes.yaml` | Branch class patterns and metadata | humans (rare) | all scripts |
| `.cascade/config.yaml` | Repo-wide knobs (version_depth, upstream remote) | humans (once) | all scripts |
| `.cascade/ownership_rules` | gitignore-style patterns for project-owned paths | humans (rare) | `check.ts`, ownership derivation |
| `.cascade/parent_branch` | Version source for an edition; only on branches that need it | humans (per edition) | `version.ts`, `check.ts` |
| `.cascade/bypass-log` | Append-only record of acknowledged CI bypasses | `cascade bypass` (humans invoke) | `check.ts` |

## Derived artifacts

| Path | Regenerated by | Gitignored |
|---|---|---|
| `/.ownership_map.txt` | `cascade ownership` | yes |
| `/.edition-snapshot.json` | edition build step | yes in source repo; shipped to deploy repos |
| Git tags `<branch>/<A.B.C.D>` | `cascade merge` / `cascade tag` | n/a |

## File schemas

### `.cascade/branch-classes.yaml`

```yaml
classes:
  - name: upstream
    pattern: '^upstream/.+$'
    read_only: true

  - name: core
    pattern: '^core$'
    base: upstream/main
    version_source: upstream/main

  - name: module
    pattern: '^module/[^/]+$'
    base: core
    version_source: core

  - name: channel
    pattern: '^channel/[^/]+$'
    base: core
    version_source: core
    co_merges: ['upstream/skill/*']

  - name: skill
    pattern: '^skill/[^/]+$'
    base: core
    version_source: core

  - name: skill-adapter
    pattern: '^skill/([^/]+)/[^/]+$'
    base_from_match: 'skill/$1'
    version_source_from_match: 'skill/$1'

  - name: module-adapter
    pattern: '^module/([^/]+)/[^/]+$'
    base_from_match: 'module/$1'
    version_source_from_match: 'module/$1'

  - name: edition
    pattern: '^edition/[^/]+$'
    base: core
    version_source: declared  # via .cascade/parent_branch

  - name: deploy
    pattern: '^deploy/[^/]+$'
    base: 'edition/*'
    version_source_from_ancestry: 'edition/*'

  - name: ephemeral
    pattern: '.*'
    fallback: true
    not_versioned: true
```

Topological order for ownership derivation follows the list order (first match wins).

**Ephemerals need special handling — tbd on actual use.** `parentOf` and `versionSourceOf` on an ephemeral branch have no mechanical answer (ephemerals can branch from anywhere; not versioned). The exact contract — return `null`, derive from `git merge-base`, throw — is deferred until a concrete consumer needs it. Noted here so it isn't forgotten during `branch-graph.ts` implementation.

### `.cascade/config.yaml`

```yaml
version_depth: 3
upstream_remote: upstream
upstream_main_branch: main
```

### `.cascade/parent_branch`

One line, the branch name of the version source:

```
core
```

Present only on `edition/<name>` branches (or on any other branch that needs to override the mechanical default).

### `.cascade/bypass-log`

Append-only. Columns: commit SHA, date, branch, bypassed rule, reason.

```
abc1234  2026-04-13  deploy/prod-acme  merge-preserve    hotfix during P1 outage; followup core-hotfix-2026-04-13
def5678  2026-04-20  edition/starter   prefix-mismatch   intentional cross-version test edition
```

`check.ts` consults this to suppress warnings on listed commits. New violations not in the log still fail.

**Accepted risk: bypass-log is trust-based.** Any committer can append an entry and suppress the corresponding warning. This is intentional — fls-claw's threat model is accidental-mistake, not adversarial-dev. Forge-level gates on bypass-log changes are disproportionate to the risk given how many other things a committer can break.

### `/.ownership_map.txt` (derived)

One line per file, sorted by path:

```
container/skills/browser/index.ts    skill/browser
container/skills/pdf-reader/index.ts skill/pdf-reader
package-lock.json                    project
src/channels/registry.ts             core
src/channels/telegram/sender.ts      channel/telegram
src/index.ts                         core
```

### `/.edition-snapshot.json` (derived, for cross-repo deploys)

```json
{
  "edition": "starter",
  "version": "1.9.0.2",
  "generated_at": "2026-04-13T10:00:00Z",
  "core_version": "1.9.0.5",
  "upstream_version": "1.9.0",
  "included": {
    "channels": ["whatsapp", "telegram"],
    "skills": ["image-vision"],
    "adapters": [{"skill": "reactions", "channel": "whatsapp"}]
  },
  "ownership_map": {
    "src/channels/telegram/sender.ts": "channel/telegram",
    "src/index.ts": "core"
  },
  "branch_classes": { "...": "copy of .cascade/branch-classes.yaml" }
}
```

Lets a cross-repo deploy run P3 locally without the full branch graph. Bundled with the edition build.
